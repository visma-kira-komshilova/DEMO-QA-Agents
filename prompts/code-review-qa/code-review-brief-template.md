**CRITICAL:** Maximum 450 words (STRICTLY ENFORCED). Single repository per report.

## Validation Rules

### Structure Check
- [ ] All 6 sections present (Header, Code Quality Review, Critical Issues, Test Coverage, Action Items, Footer)
- [ ] Code Quality Review has both Standard Checks AND Hotfix Patterns (each in `<details>`)
- [ ] All 6 Standard Checks and all repo-specific Hotfix Patterns present

### Content Quality
- [ ] All critical issues have file:line references
- [ ] Action items are specific and actionable
- [ ] Risk level correct (LOW/MEDIUM/CRITICAL)
- [ ] Word count ≤ 450

### Content Filtering Priority

When reducing to fit 450 words, remove in this order:
1. **Keep (Never Remove):** Critical issues, file:line refs, risk + recommendation, failed hotfix patterns
2. **Simplify First:** Test coverage → counts only, action items → 3-5 max, hotfix patterns → failures only
3. **Remove If Needed:** High-severity issues, test coverage section (if none missing), pattern table (if all pass)

---

## Constraints

- Max 450 words — all content counted, no exclusions
- Single repository only (no cross-repo checks)
- GitHub-flavored markdown

## Output Location

`reports/code-review/<TICKET-ID>-code-review-brief.md`
